# IP 网络常春藤韧性模型

## 概述

常春藤韧性模型源自**华为提出的 Hedera Architecture（常春藤架构）**，是借鉴常春藤植物生长特性而抽象出的网络韧性设计框架。常春藤具有**多附着点**、**网络化生长**、**局部受损不致命**、**自适应攀爬**等特征，这些生物学特性与高韧性 IP 网络的架构诉求高度一致。

2026 年 6 月，在丹麦哥本哈根举办的 DTW 26（全球数字化转型峰会）上，华为携手 TM Forum 和 GSMA 正式发布该架构，提出以**"绿色 + 韧性"为双引擎**，推动 GNI 标准与常春藤架构的广泛应用，构建"低碳可衡量、韧性可验证、演进可持续"的网络新蓝图。

该模型强调**去中心化、多点冗余、自愈能力和渐进式降级**，是对传统"主备切换"思维的升维——从单点保护的防御范式转向多点共存的网络化生存范式。

## 二、Hedera Architecture（常春藤架构）核心特征

### 2.1 绿色 + 韧性双引擎

华为明确提出：绿色网络不是"选择题"而是"必选题"，韧性是绿色网络的"及格线"，AI 是迈向 2030 零碳愿景的"加速器"。常春藤架构的三大核心支柱：

- **GNI 标准**：由 GSMA 智库主导，已覆盖 20+ 移动和固网运营商，评价体系从整网精细到单站单元
- **常春藤框架**：通过数字孪生 + AI 数据治理，推动传统无线基站向绿色智能站点转型，传统 CO 机房向边缘 DC 升级
- **AI 智能体**：推动网络从"被动节能"向"主动重构"跃迁

### 2.2 破解三角矛盾

常春藤架构旨在破解 ICT 行业长期存在的三大矛盾：

```
           业务发展
           /\
          /  \
         /    \
        /______\
  绿色降本      安全韧性
```

通过常春藤架构 + GNI 标准 + AI 智能体的结合，实现三者的协同发展。

### 2.3 生物特征 → 网络工程映射

| 常春藤特征 | 网络工程映射 | 华为关键机制 |
|-----------|-------------|-------------|
| 多点附着 | 多宿接入、多平面连接 | 双归/多归、ECMP、NetEngine 集群 |
| 藤蔓网络化 | 网状拓扑、多路径转发 | SRv6 Policy、指纹切片 |
| 局部受损存活 | 故障域隔离、自愈切换 | BGP PIC、TI-LFA、AI 自愈 |
| 自适应攀爬 | 动态路由、流量调度 | 网络数字地图 + AI 智能调度 |
| 层叠覆盖 | 多层防护、深度防御 | REA 三层架构 |
| 剪枝再生 | 快速收敛、路径重计算 | BFD + AI 预测性切换 |

### 2.4 模型四层结构

常春藤模型将网络韧性分为四层，每层对应植物结构的不同部分：

```
+---------------------------+  ← 业务层（叶）
|  业务感知 & 智能调度       |     网络数字地图 + AI 大模型
+---------------------------+  ← 控制层（茎）
|  路由决策 & 路径选择       |     iMaster NCE + AI 智能体
+---------------------------+  ← 转发层（藤蔓）
|  多路径转发 & 快速切换      |     NetEngine + SRv6/MPLS
+---------------------------+  ← 物理层（根/附着点）
|  链路/设备冗余 & 物理连接   |     NetEngine 5000E/8000 系列
+---------------------------+
```

- **物理层（根/附着点）**：NetEngine 系列路由器提供多点物理连接，单连接失效不影响整体
- **转发层（藤蔓）**：SRv6 Policy、指纹切片实现数据在多条路径间灵活流动
- **控制层（茎）**：iMaster NCE + 网络数字地图，AI 智能体动态感知状态变化
- **业务层（叶）**：面向 ToC/ToH/ToB（CHB）三大场景，AI 驱动体验保障

## 三、REA 韧性工程架构

REA 是华为韧性网络的工程实现框架，包含三个层面：

### 3.1 Robust Element（高可用网元）

确保单个网元自身的高可靠性：
- 关键部件冗余（主控板、交换网板、电源、风扇 1+1/N+1 备份）
- 控制面与转发面物理分离
- Graceful Restart / NSR（Non-Stop Routing），控制面重启转发不中断
- 软件升级不中断转发（ISSU）

### 3.2 Elastic Network（弹性分布式冗余架构）

构建网络级别的弹性能力：
- **"防得住"**：预防机制 —— 故障前通过混沌工程演练发现风险
- **"扛得稳"**：抗压机制 —— 故障发生时通过多路径、弹性冗余持续承载业务
- **"可逃生"**：逃生机制 —— 极端故障下保障关键业务最低可运行水平

弹性网络关键技术：
- ECMP + UCMP 多路径负载分担
- TI-LFA 拓扑无关快速重路由
- BGP PIC 前缀无关收敛
- SRv6 Policy 多候选路径
- 网络数字地图实现全网可视化

### 3.3 Assured Service（关键业务优先）

在资源受限或故障场景下，确保高优先级业务的服务质量：
- 优先级队列调度（PQ/LLQ）
- 指纹切片技术（基于用户 ID 免规划部署切片）
- 带宽池化 + AI VIP 流量自动识别调度
- "三零三优"目标：零单点故障、零重大事故、零中断；业务优先、用户优先、网络优先

## 四、韧性成熟度模型

将网络韧性划分为 5 个等级，对应常春藤从幼苗到成年覆盖的生长阶段：

| 等级 | 名称 | 特征 | 韧性能力 |
|------|------|------|---------|
| L1 | 种子期（Seed） | 单链路、单设备 | 无韧性，任何故障即中断 |
| L2 | 萌芽期（Sprout） | 基本冗余（主备） | 单点故障可手动切换，恢复时间分钟级 |
| L3 | 生长期（Growth） | 多路径、自动检测 | 故障自动发现，收敛时间秒级 |
| L4 | 繁茂期（Bloom） | 全网状自愈 + AI 辅助 | 故障亚秒级恢复，AI 辅助决策 |
| L5 | 常青期（Evergreen） | 自适应、自优化、可演进 | AI 预测性切换、自愈，韧性随网络规模增长不退化 |

**L5 常青期的核心能力**（华为 AI 智能体支撑）：
- 故障发生前即完成路径切换（Predictive Failover）
- 引入新设备/链路后韧性自动适配
- 网络韧性随节点数增长而增强（网络化效应）
- AI 大模型驱动的自动化根因分析与修复

## 六、韧性评估指标

| 指标 | 含义 | L5 目标 |
|------|------|--------|
| 韧性覆盖率 | 具备冗余保护的链路数 / 链路总数 | ≥ 99% |
| 故障恢复时间 | 检测时间 + 收敛时间 + 切换时间 | < 50ms（运营商级） |
| 故障影响范围 | 故障影响业务数 / 总业务数 | < 0.1% |
| 可用性 | MTBF / (MTBF + MTTR) | ≥ 99.999%（5 个 9） |
| 多路径比例 | 路径数 ≥ 3 的节点对数 / 总节点对数 | ≥ 95% |

| 维度 | 传统冗余模型 | 常春藤韧性模型（Hedera） |

## 八、落地建议

1. **从评估开始**：基于 GNI 标准和韧性成熟度模型，评估现网所处等级（L1 ~ L5）
2. **消除 SPOF**：优先解决物理层单点故障（单链路、单设备、单电源、单平面）
3. **部署 BFD + TI-LFA**：在核心/汇聚层启用毫秒级故障检测和快速切换
4. **引入网络数字地图**：实现全网拓扑可视化和自动故障定位
5. **引入 AI 智能体**：逐步从被动运维向 AI 辅助决策和预测性运维过渡
6. **构建混沌工程体系**：定期模拟故障，持续验证和提升韧性等级
7. **对标 GNI 标准**：将绿色能效与网络韧性统一考量，实现低碳 + 高可靠的协同目标
